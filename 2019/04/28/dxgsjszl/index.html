BAT公司技术晋升之路

发表于 2019-04-28 | 分类于 交流 ， 职业规划 |
阅读时长 ≈ 2

序

在中国，国内顶级互联网公司 BAT、TMD。这些公司都有一套自己的晋升规范，但是大同小异，无非是各个技术人员的打怪升级过程，就拿腾讯的来说：

腾讯级别 描述
T1 助理工程师
T2 工程师
T3 高级工程师
T4 专家
T5 科学家
T6 首席科学家

成长之路

由于笔者级别是在T3，所以笔者只能引领大家晋升到T3，后面能否升到T4，就看各位的造诣和机会。

路线图

graph LR
    T1.1 -->T1.2
    T1.2 -->T1.3
    T1.3 -->T2.1
    T2.1 -->T2.2
    T2.2 -->T2.3
    T2.3 -->T3.1
    T3.1 -->T3.2
    T3.2 -->T3.3

大家看到这么漫长的晋级之路，千万不要气馁：小级之内，比较容易升，一般半年升级一次。大级来说就相对比较麻烦，T3 一个坎， T4一个坎。

如何成为优秀的T1 - 学习

基础知识 系统设计 工作方法
xx 编程思想、基础 进程模块划分 项目管理
xx 多线程、并发 核心数据结构 研发流程
Effective xx 典型业务流程 时间管理
Mysql UML

如何成为优秀的T1 - 实践

编码质量 学以致用 权衡设计
遵循编码规范 编码 掌握现有模块设计和实现
使用规范工具 调试 思考不合理和合理的地方
斟酌代码 运维 重构思路
解决问题

如何从T2到T3 —学习

书籍 系统设计 工作方法
《高效能人士的七个习惯》 可靠性设计 方法论建设
《大型分布式网站架构设计与实践》 通用性设计 总结、分享
<<分布式系统：概念与设计>> 安全性设计
<<操作系统——精髓与设计原理>> 高性能设计
<<应用密码学>> 容灾设计

如何从T2 到T3 —实践

逐步负责 让人放心 能力
各种Server 设计 精通自己设计的系统 技术全局观和部门内的技术影响力
通信模型、协议设计 自信、担当 独立/带领团队解决疑难问题，锻炼问题分析与解决的能力
熟悉各类机型、硬件性能指标、适用场景 尝试掌控 带领项目，锻炼项目管理能力
端到端的系统设计及实现，从业务需求到资源准备

总结

如何获得成长：

  • 学习（培训）获得机会，进入好的平台（前置条件）。
  • 在好的平台中，在实践中成长（70%）。
  • 自学（20%）
  • 培训 （10%）
